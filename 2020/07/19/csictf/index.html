Pwn-2

This problem provides a binary pwn-2. Using ghidra we can see the main function has an unchecked buffer followed by a declared int local_c.

The idea is to overrun local_38 and write to local_c. It’s little ending so the values need to be put in reverse.

* Note: Ghidra had the needed value as ‘-0x35014542’ but this didn’t work. Using hopper I was able to find the actual value needed ‘0xcafebabe’. End the end both tools provided correct and incorrect information.

Payload:

Flag: csictf{c4n_y0u_re4lly_telep0rt?}

Unseen

With his dying breath, Prof. Ter Stegen hands us an image and a recording. He tells us that the image is least significant, but is a numerical key to the recording and the recording hides the answer. It may seem as though it’s all for nothing, but trust me it’s not.

* I did not solve this in time but wanted to put this here as a reminder. I knew everything needed to solve this problem but failed to put the pieces together in time. The official writeup is at https://github.com/csivitu/ctf-challenges/tree/master/forensics/unseen

This problem provides a moris.wav and nyc.png file.

using stegsolve I was able to extract the key from the image.

key: 42845193

The audio file is morse code. Using https://morsecode.world/international/decoder/audio-decoder-adaptive.html I got the read out as SEARCH-SOMEWHERE-ELSE. This message is both helpful and not (?).

The visual form of the message is:
... . .- .-. -.-. .... -....- ... --- -- . .-- .... . .-. . -....- . .-.. ... .
111 1 10 101 0101 1111 011110 111 000 00 1 100 1111 1 101 1 011110 1 1011 111 1

Next run steghide against moris.wav and use the password ‘42845193’

The flag file looks empty but looking at it in hex we can see it’s filled with tabs and spaces.
